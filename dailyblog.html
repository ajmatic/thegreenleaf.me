August 9

My main page is The Green Leaf. And I have just added a link to that page by putting a hyperlink tag to it. That hyperlink tag looks a little something like this:
<a href="index.html" id="tag">
Now you can click on the link in this paragraph and go to my homepage. The id=tag is not necessary unless you want to do something different with the link. I wanted to make this particular link the color green, and I did that by declaring it in a css file. That's where the id=tag comes in handy.

I reference the hyperlink in my css file by putting an octothorpe (#) in front of the word tag. My css code looks something like this.
#tag {color: green;}
What I've done is referenced my html code by giving the hyperlink an id and declaring that I want the color of that id to be green.

If I did not link up my html file with a css file (which I'll show you how to do later) the display of the link would set back to the browser's normal default setting, which would look this: The Green Leaf


August 15

Ok, so I suppose we should start at the beginning. In order to start coding you're going to need a text editor on your computer. I use Sublime Text, it's easy to find and download, and it's free. Once you download that (and make sure you download it for the computer you are using, i.e. - Windows, Mac) you can start coding. When you open Sublime Text you can start your first html file. A good standard practice is to call your first html file index.html, this will essentially be your homepage and when you want to put it online, your host (the server that will make your site live) will usually require that you have a file named index. Check out NameCheap to browse for domain names and they will also be able to host your site.

So, once you have your text editor you're ready to begin. All html documents must start off with the declaration
<!doctype html>
This begins your document and tells the web browser what version of html the page is written in. To understand more on this particular element go here

Your first tag will be an <html> tag.
This will have a closing tag to it. It will look like this </html>
This tag will encompass everyting within your html file. Your <body> tag, paragraph <p> tags, and all other elements. Whenever you open an element you will use these less-than and greater-than signs <> Ex. - <html> or <body>

When you want to close an element you will use the same signs but add a backslash after the less-than symbol. Ex - </html> or </body>

There are many different elements, for a complete list you can go here.

It's a little overwhelming, I know! But once you get familiar with all the different element tags, you will be able to do so much. We will slowly go through the majority of the elements and what they mean, but if you want to get a head start on creating your own website, w3schools is a great place to start.

Ok, your homework for today is to download a text editor and to check out w3schools. I'm going to learn a little more myself today and will be back tomorrow to go over the head of an html document. Until then....

August 16

Ok, so after you have downloaded your text editor, and declared your doctype, you may put in the head of an html page. What mine usually consist of are several things, first you need a <head> opening tag and a </head> closing tag. Put a few lines between the opening and closing tags. After the opening tag on another line and indented in, you want to put your meta tag. This will provide metadata about the html document but will not show up in the webpage. It's merely a communication of sorts, to learn more click here.

Then you want to put your <title> tag, this will show up in the tab that opens on the top of your computer screen. Next you need your link to a stylesheet. Any other links for internal purposes will go here. For example, if you want to use a cool font style, you may link to a google font page and more specifically to a certain font style. Overall, the top of your html document should look something like this:

<head>
<meta charset="utf-8" />
<title>The Green Leaf</title>
<link rel="stylesheet" href="css.index.css />
<link href='http://fonts.googleapis.com/css?family=Felipa' rel='stylesheet' type='text/css'>
</head>

Learn a little more about the attributes used in the head section by checking w3schools. And I'll be back tomorrow so we can start on the body and do some headings.

August 17

Ok, so today I'm going to go over the body tag and headings. The body tag is quite simple. It will come after your head tag and will encompass all the content on your document that will eventually appear on your webpage. Body tags look like this:
<body></body>

Headings are, just like the name implies, headings. This is the content that will standout to whoever looks at your webpage. There are 6 heading types. <h1> is the biggest heading and it also carries the most weight to it. What that means is when your site gets catalogued online, the h1 can give you Search Engine Optimization or simply, SEO. Here is a tutorial all about h1 tags and SEO. <h1> is the biggest heading while <h6> is the smallest and has the least weight. You can open an h1 and close it like this
<h1>Enter Any Text Here!!</h1>
You can have as many different headings as you like. Positioning all of them will come later.

Maybe you can try something like this but with your own name:
<h1>Hello World!</h1>
<h2>My name is Adam.</h2>
See what this looks like
Or you can try it yourself on your own text editor, if you haven't downloaded one yet, try this!

So get a little comfortable with headings, I've got a boat ton of work to do, and I'll be back Monday. That's when we'll work on putting together images, paragraphs and other elements to fill out your webpage.

August 23

Ok, it's been a few days and I haven't posted yet. Well there is a reason for that. I have just found out the hard way that you really need to link in resets to your html pages. CSS resets are imperative. Their purpose is to zero everything out so you can start your webpage with basically a clean canvas. Not all people who go online use the same browser. Some use Chrome, some use Firefox. Others use Safari, and those who don't know any better use Internet Explorer, lol! No, I'm kidding, I don't mean to make fun. But my point is that each of those browsers have different default settings that affect the look of your page when it's opened in that particular browser. Certain browsers will have the default font set at serif, while others will have it as Times New Roman. In order to have your webpage look the same across the board, you are going to have to link up your html page to an external css reset file. I have been using Eric Meyer's reset. Just copy Eric Meyer's reset, it's free, and paste it in a CSS file on your computer. Then you can put the link to the reset in your head element, put it below the title tag and above the link to your other stylesheet. That way it will zero everything out and it will allow you to style it the way you want to. There are other resets out there. So please feel free to do some research. You might want to start here when researching different resets.
